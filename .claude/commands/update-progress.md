# update-progress

태스크 완료 후 PROGRESS.md와 PLAN.md의 상태를 업데이트합니다.
에이전트가 직접 파일을 수정하는 대신 이 커맨드를 사용해 표준 형식을 유지합니다.

**사용법:** `/update-progress [태스크ID] [상태]`
- 상태: `done` | `failed` | `started`
- 예시: `/update-progress P1-T1 done`

**실행 절차:**

1. PLAN.md에서 해당 태스크 행의 상태를 업데이트하라:
   - `started` → 행의 `⬜ 미시작`을 `🔄 진행중`으로 변경
   - `done` → 행의 `🔄 진행중`을 `✅ 완료`로 변경
   - `failed` → 행의 상태를 `❌ 실패`로 변경

2. PROGRESS.md를 다음 섹션에 맞게 업데이트하라:

**완료(`done`)인 경우:**
```
## 완료된 태스크 테이블에 행 추가:
| [태스크ID] | [현재날짜 YYYY-MM-DD] | [role명] | [작업 결과 한 줄] |

## 에이전트 세션 로그에 추가:
[날짜 시각] <role명> @ [태스크ID]: [완료 내용 한 줄] → 성공
```

**실패(`failed`)인 경우:**
```
## 차단된 태스크 테이블에 행 추가:
| [태스크ID] | [실패 이유] | agent-optimizer 실행 필요 |
```

3. 업데이트 완료 후 다음을 출력하라:
```
✅ PROGRESS.md 업데이트 완료
- 태스크 [태스크ID]: [이전상태] → [새상태]
- 다음 미시작 태스크: [PLAN.md에서 조회한 다음 태스크]
```
