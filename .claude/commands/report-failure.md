# report-failure

태스크 실패 시 표준 형식의 실패 보고서를 생성합니다.
Antigravity가 `agent-optimizer`를 실행하는 데 필요한 정보를 구조화합니다.

**사용법:** `/report-failure [태스크ID]`

**실행 절차:**

1. 다음 정보를 수집하라:
   - 가장 최근의 에러 메시지 (정확한 텍스트)
   - 에러가 발생한 파일과 라인 번호
   - 시도한 해결 방법

2. 다음 형식으로 실패 보고서를 작성하라:

```markdown
## 실패 보고서

**태스크 ID**: $ARGUMENTS
**Role**: [frontend_dev / api_dev / test_verify / devops / security_auditor]
**실패 시각**: [현재 시각]

### 에러 내용
\`\`\`
[정확한 에러 메시지]
\`\`\`

### 에러 위치
- 파일: [파일 경로:라인번호]
- 단계: [빌드 / 테스트 / 로직 오류 / 배포 / 기타]

### 시도한 해결책
1. [시도 1]
2. [시도 2]

### 실패 분류 추정
- [ ] C1: 컨텍스트 부재 (SOP에 관련 지침 없음)
- [ ] C2: 가치 위반 (Meki 핵심 가치 위배 코드)
- [ ] C3: 스코프 혼동 (담당 외 파일 수정)
- [ ] C4: 검증 부재 (테스트 미실행으로 뒤늦게 발견)
- [ ] C5: 프롬프트 모호 (태스크 설명 불명확)

### SOP 개선 제안
[어떤 규칙을 어느 SOP에 추가하면 이 실패를 방지할 수 있는가]
```

3. 보고서 작성 후 더 이상 작업을 진행하지 마라. Antigravity가 `agent-optimizer`를 실행한 후 재지시를 기다려라.
