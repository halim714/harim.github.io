# Agent Improvement Protocol

이 워크플로우는 스웜 에이전트(Claude Code)가 실패하거나, Antigravity 오케스트레이터가 생성된 코드에서 문제를 감지했을 때 **반드시** 실행해야 합니다.

> **핵심 원칙**: 코드를 고치는 것은 증상 치료다. 룰과 워크플로우를 고치는 것이 근본 치료다.

## 언제 이 워크플로우를 실행하는가

- 스웜 에이전트가 동일한 유형의 오류를 반복할 때
- 생성된 코드가 Meki의 핵심 가치(데이터 주권, 사유 흐름, 위키 연결성)에 위배될 때
- 에이전트가 잘못된 가정으로 작업을 진행했을 때 (e.g. 잘못된 API 버전, 존재하지 않는 파일 경로 참조)
- `swarm-manager` 워크플로우 Step 4(검증)에서 문제가 발견되었을 때

---

## Step 1: 실패 분류 (Root Cause Analysis)

실패 원인을 다음 중 하나로 분류하라:

| 분류 | 설명 | 예시 |
|---|---|---|
| **C1: 컨텍스트 부재** | 에이전트가 프로젝트 구조나 제약을 몰랐음 | GitHub API 버전 오인, 잘못된 import 경로 |
| **C2: 가치 위반** | Meki 핵심 가치에 반하는 구현 | 원본 데이터 수정, 서드파티 서버에 데이터 전송 |
| **C3: 스코프 혼동** | 여러 파일을 동시에 수정하여 충돌 발생 | 두 에이전트가 같은 파일 수정 |
| **C4: 검증 부재** | 에이전트가 자기검증(테스트) 없이 완료 보고 | npm test 미실행, 브라우저 검증 생략 |
| **C5: 프롬프트 모호** | 지시가 너무 넓어 에이전트가 추측으로 구현 | "AI 기능 개선해줘" 류의 지시 |

## Step 2: 룰 생성 또는 업데이트 (Harness Fix)

분류 결과에 따라 `.agents/rules/` 에 새 파일을 생성하거나 기존 파일을 업데이트하라.

**파일 네이밍 규칙**: `{분류코드}-{주제}.md` (예: `c1-github-api-constraints.md`)

**룰 파일 형식**:
```markdown
---
trigger: always_on
description: [이 룰이 방지하는 실패 유형 한 줄 요약]
---

# [룰 제목]

## 컨텍스트
[왜 이 룰이 필요한지 — 어떤 실패에서 비롯되었는지]

## 규칙
- [구체적이고 행동 가능한 지시 1]
- [구체적이고 행동 가능한 지시 2]

## 올바른 예시
[코드 또는 행동 예시]

## 잘못된 예시
[에이전트가 저지른 실수 예시]
```

## Step 3: 워크플로우 업데이트 (Workflow Fix, 필요 시)

실패가 룰이 아닌 **워크플로우 설계 문제**에서 비롯되었다면:
1. `swarm-manager.md` 또는 해당 워크플로우 파일을 직접 수정하라
2. 수정 내용을 커밋 메시지에 명시: `chore(agents): fix workflow - [실패 유형]`

## Step 4: 자기검증 후 재실행 (Self-Correction)

룰/워크플로우를 업데이트한 뒤:

```bash
# 새 룰이 적용된 상태에서 실패했던 태스크를 재실행
./scripts/run-swarm.sh <모델> "<원래 실패한 태스크 프롬프트 + 새 룰 파일명 명시>" /Users/halim/Desktop/meeki/meki/miki-editor <식별자>-retry
```

## Step 5: 검증 및 보고 (Verification & Report)

재실행 후:
1. `git diff`로 변경 내용 확인
2. 가능하면 `npm run test` 또는 관련 테스트 실행
3. 사용자에게 다음 형식으로 보고:

```
## Agent Improvement Report

**실패 분류**: C1 / C2 / C3 / C4 / C5
**실패 원인 요약**: [한 문장]
**생성된 룰 파일**: `.agents/rules/[파일명].md`
**수정된 워크플로우**: (해당 시) `.agents/workflows/[파일명].md`
**재실행 결과**: 성공 / 실패 (실패 시 이유)
**다음 권장 조치**: [사용자가 승인해야 할 사항]
```
