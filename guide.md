# "미키" AI 위키 사용자 가이드 (v5 - AI 제안 및 승인 워크플로우)

"미키"는 복잡한 위키 문법이나 마크다운을 전혀 몰라도, 마치 대화하듯 자연어로 AI에게 요청하여 글을 쓰고 위키 기능을 활용할 수 있도록 돕는 혁신적인 AI 기반 위키 시스템입니다. 이 최신 버전(v5)은 사용자님의 피드백을 적극 반영하여, AI가 명령을 자동 실행하는 대신 **사용자에게 "제안"하고 사용자의 "승인"을 통해 적용하는 새로운 워크플로우**를 도입했습니다. 최신 AI 에디터 트렌드에 따라 사용자 제어권을 강화하고 보다 직관적인 인터랙션을 제공하며, 모든 편집 결과는 여전히 WYSIWYG로 즉시 확인하고 AI 편집 작업에 대한 Undo/Redo까지 완벽하게 지원합니다. Jekyll과 GitHub Pages를 기반으로 구축되어 무료로 호스팅할 수 있습니다.

## 미키 에디터 사용법 (v5 - AI 제안 및 승인 워크플로우)

미키 에디터 v5의 핵심은 **AI의 제안과 사용자의 승인**을 기반으로 하는 새로운 편집 워크플로우입니다. `https://username.github.io/editor.html` (또는 로컬 실행 시 해당 주소)로 접속하여 사용합니다.

### 인터페이스 소개

-   **왼쪽: 에디터 패널**: WYSIWYG 편집 영역입니다. 이곳에서 자연어로 AI에게 명령을 내리면, AI의 제안을 받고 승인하여 편집을 진행합니다.
-   **오른쪽: AI 상호작용 패널**: AI와의 직접적인 대화, 프롬프트 입력, 지식 그래프 확인 등 보조적인 AI 기능을 제공합니다. 주요 편집은 에디터 내 AI 제안 및 승인 워크플로우를 통해 이루어집니다.

### 자연어 명령 입력 및 AI 제안 받기

더 이상 `/ai` 같은 특별한 접두사를 사용할 필요가 없습니다. 에디터에 일반적인 문장으로 원하는 작업을 지시하세요.

1.  **명령 입력**: 에디터의 빈 줄이나 기존 텍스트 줄에서 다음과 같이 자연스럽게 명령을 입력합니다:
    *   `이 부분을 굵게 만들어줘`
    *   `다음 내용으로 순서 없는 목록을 만들어줘: 사과, 바나나, 체리`
    *   (텍스트를 드래그하여 선택한 후) `이 선택 영역을 인용문으로 바꿔줘`
2.  **Enter 키 입력 또는 컨텍스트 변화 감지**: 명령을 입력한 줄에서 Enter 키를 누르거나, 특정 컨텍스트(예: 텍스트 선택 후 특정 키 입력 등) 변화 시 시스템은 해당 입력을 잠재적 명령으로 감지합니다.
3.  **컨텍스트 자동 인식 및 AI 전달**: 시스템은 사용자의 입력뿐만 아니라, 현재 커서 위치, 드래그하여 선택한 텍스트 영역, 명령이 입력된 줄의 내용, 그리고 문서 전체의 내용을 종합적으로 파악하여 AI(현재는 Mock AI)에게 전달합니다.
4.  **AI의 구조화된 제안 생성**: 사용자의 자연어 명령을 받은 AI(Mock AI)는 분석된 내용을 바탕으로 **구조화된 JSON 형식의 제안**을 생성합니다. 이 제안 JSON에는 다음과 같은 정보가 포함됩니다 (개념적 예시):
    ```json
    {
      "isSuggestion": true,
      "suggestionId": "unique-suggestion-id",
      "suggestionType": "user_command_transform", // or "ai_proactive_suggestion"
      "displayText": "선택한 텍스트를 굵게 표시할까요?", // 사용자에게 보여줄 제안 문구
      "actions": [ // 실제 실행될 작업 지시 배열 (v4의 실행 명령과 유사)
        {
          "actionType": "apply_style",
          "commandType": "bold",
          "range": [[라인, 문자], [라인, 문자]] // 적용 범위
          // ... 기타 필요한 파라미터
        }
      ]
    }
    ```

### AI 제안 UI 및 사용자 승인/취소

AI가 제안을 생성하면, 에디터 내 적절한 위치(보통 명령이 입력된 곳 근처 또는 제안 대상 텍스트 근처)에 **풍선(popover) 형태의 제안 UI**가 나타납니다.

1.  **제안 확인**: 제안 UI에는 AI가 제안하는 내용(`displayText`)이 표시됩니다. 예를 들어, "이 부분을 표로 바꿀까요?" 또는 "선택한 텍스트에 링크를 추가할까요?" 와 같이 나타납니다.
2.  **승인 (Run)**: 제안 내용이 마음에 들면 **"적용(Run)"** 버튼을 클릭합니다. 그러면 시스템은 해당 제안의 `actions`에 정의된 작업을 다음 단계에서 에디터에 실제로 적용합니다.
3.  **취소 (Cancel)**: 제안을 원하지 않으면 **"취소"** 버튼을 클릭합니다. 제안 UI는 사라지고 에디터에는 아무런 변경도 일어나지 않습니다.

### 승인된 제안의 자동 WYSIWYG 적용

사용자가 제안 UI에서 "적용(Run)" 버튼을 클릭하면, AI 제안 JSON의 `actions` 배열에 포함된 작업 지시들이 실행됩니다.

-   **자동 WYSIWYG 적용**: 에디터(`MikiEditor.js`의 `applyStructuredAiCommand` 함수)는 이 작업 지시들을 해석하여, **Toast UI Editor의 직접적인 WYSIWYG API를 사용해 변경사항을 자동으로 적용**합니다. 사용자는 마크다운 코드를 전혀 볼 필요 없이, 편집 결과가 즉시 시각적으로 반영되는 것을 경험하게 됩니다.
-   지원되는 작업은 텍스트 서식, 블록 요소, 링크, 표, 목록 등 다양합니다.

### AI 편집 실행 취소/다시 실행 (Undo/Redo)

AI 제안을 승인하여 적용된 편집 내용이 마음에 들지 않거나 실수로 승인했을 경우, 표준 단축키를 사용하여 AI 편집 작업을 되돌리거나 다시 실행할 수 있습니다.

-   **실행 취소 (Undo)**: `Ctrl+Z` (Windows/Linux) 또는 `Cmd+Z` (Mac)
-   **다시 실행 (Redo)**: `Ctrl+Y` (Windows/Linux) 또는 `Cmd+Shift+Z` (Mac)

이 기능은 AI 제안 승인을 통해 적용된 모든 편집 작업에 대해 작동하며, 편집 전후의 상태를 기록하여 복원을 지원합니다.

## AI 기능 심층 활용

### 다양한 자연어 명령 및 AI 제안 예시

-   **명령**: `이 문장 굵게 해줘`
    *   **AI 제안 UI**: "이 문장을 굵게 표시할까요? [적용] [취소]"
    *   **승인 시**: 해당 문장이 즉시 굵게 표시됩니다.
-   **명령**: (텍스트 "구글" 선택 후) `여기에 https://google.com 링크 걸어줘`
    *   **AI 제안 UI**: "선택한 '구글'에 https://google.com 링크를 추가할까요? [적용] [취소]"
    *   **승인 시**: "구글" 텍스트에 해당 URL 링크가 즉시 적용됩니다.
-   **명령**: `이름, 나이, 도시 컬럼으로 표 만들어줘`
    *   **AI 제안 UI**: "'이름, 나이, 도시'를 헤더로 하는 3x2 표를 생성할까요? [적용] [취소]"
    *   **승인 시**: 해당 헤더를 가진 빈 표가 즉시 에디터에 삽입됩니다.

### 능동적인 AI 제안 활용하기 (제안 및 승인 방식)

AI는 사용자가 글을 쓰는 동안 문서의 패턴을 분석하여 유용하다고 판단되는 편집 제안을 능동적으로 합니다. 이러한 제안 역시 에디터 내에 풍선 UI 형태로 나타나며, 사용자는 "적용(Run)" 또는 "취소" 버튼으로 상호작용할 수 있습니다.

-   **예시 제안**: "문서가 길어지고 있는데, 현재 제목들을 바탕으로 목차를 추가할까요? [적용] [취소]"
-   **승인 시**: AI가 문서 내용을 분석하여 적절한 위치에 목차를 WYSIWYG 형태로 삽입합니다.

## 문제 해결 및 중요 참고사항

-   **AI 기능이 예상대로 완벽하게 작동하지 않음 (가장 중요!)**:
    현재 버전(v5)의 핵심은 **AI가 생성한 제안을 사용자가 승인하면, 해당 작업 지시가 에디터에 WYSIWYG로 자동 적용되는 새로운 워크플로우를 구현**한 것입니다. 하지만, `AiPanel.js`의 `getAiResponse` 함수 내에서 자연어 명령을 이해하고 구조화된 제안 JSON(특히 `displayText`와 `actions`)을 생성하는 부분은 여전히 **매우 정교하게 만들어진 Mock AI(시뮬레이션)**입니다. 이 Mock AI는 다양한 예시 명령과 제안 시나리오를 처리하도록 설계되었으나, 실제 인간 수준의 자연어 이해 능력이나 모든 상황에 대한 완벽한 대처는 불가능합니다.
    **미키 AI 위키의 진정한 잠재력과 최고의 정확도를 경험하시려면, 이 Mock AI 부분을 실제 강력한 LLM(Large Language Model, 예: OpenAI의 GPT 시리즈, Anthropic의 Claude 시리즈 등) API와 연동해야 합니다.** 현재 코드는 이러한 실제 AI 모델 연동을 위한 매우 견고하고 잘 정의된 인터페이스(구조화된 제안 JSON 입출력)를 제공하므로, 연동 작업이 비교적 용이할 것입니다.
